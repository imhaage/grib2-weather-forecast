# Home View Tabs Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Split the home view into two tabs — "Visualize a full forecast run" (model list) and "Inspect a GRIB2 file" (drop zone) — with no upload state persistence on tab switch.

**Architecture:** Extend the existing `showView()` pattern with a `showTab(name)` function. HTML restructuring wraps each section in a named panel div; CSS controls visibility with an `.active` class; JS wires click handlers at init and resets upload state on every switch back to the model tab. Switching to the home view (via `route()`) always resets to the model tab.

**Tech Stack:** Vanilla HTML/CSS/JS. No new dependencies.

### Task 3: JS — Add showTab, resetUploadState, wire buttons

**Files:**
- Modify: `apps/visualize/index.js`

- [ ] **Step 1: Add `showTab` and `resetUploadState` after `showView`**

`showView` is at line 1857. Immediately after its closing `}` (line 1860), insert:

```js
function showTab(name) {
	for (const panel of ["model", "upload"]) {
		document.getElementById(`tab-panel-${panel}`).classList.toggle("active", panel === name);
	}
	for (const btn of document.querySelectorAll(".tab-btn")) {
		btn.classList.toggle("active", btn.dataset.tab === name);
	}
	if (name === "model") resetUploadState();
}

function resetUploadState() {
	byId("file-summary").style.display = "none";
	byId("results").style.display = "none";
	const status = byId("status");
	status.textContent = "";
	status.className = "";
}
```

`byId` is defined at the top of `index.js` as `const byId = (id) => document.getElementById(id)`.

- [ ] **Step 2: Call `showTab("model")` from `route()` for the home view**

In `route()` (around line 1895), the final `else` branch currently reads:

